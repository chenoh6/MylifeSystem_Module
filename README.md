---

## 痛点：你为什么要看懂代码，却总被语法拦住？

想象你在读一段代码：

```css
body {
  font-family: "Inter", -apple-system, BlinkMacSystemFont, sans-serif;
  background: #f0f4f8;
  padding: 20px;
}
```

你知道 `body` 是什么吗？`font-family` 在干嘛？`padding` 又是什么？

如果你不能在 3 秒内回答这些问题——

**你不是笨，你只是缺少一个"中间层"。**

---

## 问题到底是什么？

编程语言和人类语言之间存在一道鸿沟：

| 你看到的 | 你的大脑想要的 |
|----------|--------------|
| `def` | "定义一个函数" |
| `padding: 20px` | "元素内部四周留20px空白" |
| `SELECT * FROM users WHERE age > 18` | "从用户表里找出所有年龄大于18的人" |
| `try...except` | "尝试执行，出错了就这样处理" |

每次遇到陌生语法，你都要：
1. 搜索关键词含义
2. 理解它在上下文中的作用
3. 拼凑出完整语义

**这三步，每一步都在消耗你的注意力。**

阅读 10 行代码，你可能要打断 3 次去查语法。这不是效率问题——这是认知负担问题。

## 怎么做到的？

### 三层结构，从词到句全覆盖

#### 第一层：关键词级 —— "这个词是什么"

| 代码 | 翻译 |
|------|------|
| `def` | 定义一个函数 |
| `SELECT` | 查询/读取 |
| `padding` | 内边距 |
| `async/await` | 异步等待 |
| `JOIN` | 表连接 |

**快速定位，秒级理解。**

#### 第二层：模式级 —— "这段代码在做什么"

| 代码 | 翻译 |
|------|------|
| `def add(a, b): return a + b` | "定义函数add，把a和b传入，做加法后返回结果" |
| `items.map(x => x * 2)` | "对数组每个元素乘以2，组成新数组" |
| `SELECT city, COUNT(*) FROM users GROUP BY city` | "按城市分组，统计每个城市有多少人" |
| `body { padding: 20px; }` | "选择body标签，内部四周留20px空白" |

**不是背语法，是读句子。**

#### 第三层：完整代码翻译 —— "整段代码一句话讲清楚"

```css
.card {
  width: 300px;
  padding: 20px;
  border-radius: 8px;
  box-shadow: 0 2px 8px rgba(0,0,0,0.1);
}
```

> 定义一个卡片组件：宽度300px，内边距20px，圆角8px，带一层浅色阴影。

**一段代码，一句话，秒懂。**

## 覆盖5大语言

### CSS — 样式与布局
- **选择器**：`body`、`.class`、`#id`、`:hover` → 选谁
- **属性分类**：字体、盒子模型、背景、Flex布局、转换动画
- 按功能归类，不再东找西找

### HTML — 结构与标签
- **属性分类**：文本属性、资源属性、交互属性、样式属性
- `src` → 资源来源，`onclick` → 点击事件，`alt` → 替代文字

### Python — 逻辑与数据
- **8大分类**：变量/类型、函数、流程控制、类、列表/字典、异常、文件、导入
- `for i in range(5):` → 从0数到4
- `with open() as f:` → 自动管理文件开关

### JavaScript — 交互与逻辑
- 变量声明（let/const）、箭头函数、数组方法（map/filter/reduce）
- DOM操作、事件处理、异步（async/await/Promise）
- `await fetch(url)` → 发起请求，等待响应

### SQL — 数据库查询
- 查询（SELECT/WHERE/GROUP BY）、增删改（INSERT/UPDATE/DELETE）
- 表连接（JOIN）、子查询、索引约束
- `LEFT JOIN ... WHERE ... IS NULL` → 找出没有关联的记录

## 产品理念

> **"好的工具不应该让你学会新东西，而是让你用已有的东西解决新问题。"**

CodeSpeak 不教你写代码，它帮你**读懂代码**。

就像双语对照阅读——左边是原文，右边是翻译。你不需要精通英语也能理解内容。

**代码也一样——你不需要精通语法，也能理解它在做什么。**

## 视觉设计

本产品的视觉设计遵循 **Chromatic Syntax** 理念：

- **冷色调区域**（深靛蓝、石墨色）：代表代码的系统性与精确性
- **暖色调区域**（琥珀色、奶油色）：代表自然语言的温暖与直觉
- **珊瑚金色交界**：代表翻译发生的瞬间——语法转化为意义的时刻
